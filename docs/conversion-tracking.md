# Tracking First Call to Recharge

GitHub traffic should lead developers through a short path:

```text
GitHub repo -> first API call -> working example -> console recharge -> production workflow
```

## Suggested UTM Links

- Repo homepage: `utm_source=github&utm_medium=repo`
- Docs: `utm_source=github&utm_medium=docs`
- Examples: `utm_source=github&utm_medium=example`
- Issues: `utm_source=github&utm_medium=community`

## Events to Track

- GitHub repo visit.
- Website signup.
- API key created.
- First successful API call.
- First recharge.
- Second recharge or production usage.

## README Call to Action

Keep the call to action simple:

```text
Run your first API call. If it works, recharge $5 and get $5 extra credits.
```
